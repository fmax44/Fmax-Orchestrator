# Architecture Decision Log

## ADR-001: Использовать файловую очередь задач

### Решение

Задачи и отчёты хранятся как markdown-файлы в `.codex/tasks` и `.codex/reports`.

### Причина

Файлы легко читать человеку, удобно проверять через Git и просто использовать без отдельной базы данных.

### Последствия

Codex Desktop может работать с задачей напрямую в проекте, а ChatGPT получает прозрачную историю работы.

### Ограничения

Файловая очередь не решает конкуренцию нескольких исполнителей и требует дисциплины при ручных изменениях.

## ADR-002: Не запускать Codex Desktop автоматически в MVP

### Решение

Orchestrator не запускает Codex Desktop сам.

### Причина

MVP должен быть простым, контролируемым и безопасным. Ручной запуск уменьшает риск неожиданных действий.

### Последствия

Пользователь явно видит, когда Codex выполняет задачу.

### Ограничения

Процесс требует ручного шага между постановкой задачи и отчётом.

## ADR-003: Сделать Git обязательным

### Решение

Рабочий проект должен быть Git-репозиторием до использования diff/review workflow.

### Причина

Git даёт проверяемый diff, историю commit и понятную границу изменений.

### Последствия

`inspect_diff`, approve и commit-review становятся надёжными.

### Ограничения

Проекты без Git нужно сначала инициализировать.

## ADR-004: Не трекать `.codex` в Git

### Решение

`.codex/` по умолчанию добавляется в `.gitignore`.

### Причина

Очередь задач, отчёты и решения являются локальной рабочей координацией, а не частью продукта.

### Последствия

В Git остаются только изменения продукта, а не служебный шум.

### Ограничения

Если команде нужна общая очередь задач, это решение придётся пересмотреть.

## ADR-005: Использовать TypeScript для MCP-сервера

### Решение

MCP Orchestrator реализован на TypeScript и Node.js.

### Причина

Официальный MCP SDK доступен для TypeScript, а строгие типы помогают держать tools и сервисы предсказуемыми.

### Последствия

Сервер легко расширять новыми tools, prompts и services.

### Ограничения

Нужны Node.js и npm-зависимости.
